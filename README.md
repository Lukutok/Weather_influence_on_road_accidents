# Влияние погодных условий на ДТП
**Цель**: собрать и структурировать в базе данных сведения о ДТП и погодных условиях. Проанализировать взаимосвязь между параметрами погоды и частотой ДТП (степенью тяжести).

**География данных**: г. Екатеринбург, г. Пермь.

**Период сбора данных**: 2015-2026 гг

**Стек**:
- Python: requests, pandas, supabase, json, numpy, datetime
- Supabase: PostgreSQL
- BI система: Power BI

## 1. Сбор данных
Первым этапом через Open-Meteo API (Historical Weather API) собираем погодные почасовые данные для выбранных городов.

Из Wikipedia выгружаем информацию о всех городах России в датафрейм, присоединяем к каждому городу широту и долготу локации. Затем собираем таблицу с кодом ОКАТО и кодом регионов для получения информации о ДТП.

Через API ГИБДД собираем данные о дорожных происшествиях. Нормализуем исходные данные на 4 взаимосвязанные таблицы - общая информация о ДТП, описание места происшествия, транспортные средства и участники.

## 2. Работа с базой данных 
Для всех данных создаем таблицы на Supabase и выгружаем туда всю имеющуюся информацию.
Схема базы данных [database](https://github.com/Lukutok/Weather_influence_on_road_accidents/blob/main/database.md/ "схема БД")

## 3. Обновление данных
Каждый день через GitGub actions происходит добавление погодных данных за предыдущий день на Supabase

Каждый месяц происходит обновление таблицы ДТП - выгрузка новых данных в базу данных(в процессе реализации)

## 4. Анализ данных
Анализ данных в Jupyter Notebook и построение дашборда в Power BI (в процессе)
